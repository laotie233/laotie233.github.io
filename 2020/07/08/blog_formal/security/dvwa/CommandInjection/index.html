Command Injection

简介

​ 任意命令执行，由于网站后台没有对参数进行严格效验，并且调用 exec 等系统执行函数，造成攻击者可以执行任意后台命令。

Low级别

源码审计

  • 代码中没有对输入进行任何效验

解答

源码审计

  • 只对 && ；两个字符过滤, & | 等危险字符黑名单不过滤，仍然可被绕过

解答

High级别

源码审计

  • 相对比较严格的黑名单
  • 看了好久，发现 | 的过滤多了一个空格(wc)
  • 开发者通过黑名单的方式防止任意命令很难没有失误

解答

通过这个 | 带空格绕过

Impossible级别

欣赏源码

  • 白名单检测，检测ip
  • 检测ip是不是4个段 + 每个段是不是数字
  • 白名单检测才是防止攻击者入侵的正确解法

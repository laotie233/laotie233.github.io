Brute Force

简介

​ 暴力破解。相对简单而常用的攻击方式。通俗来讲就是攻击者通过枚举密码字典，暴力破解鉴权服务，本文已DVWA为例，介绍各个等级的攻击方式和保护方式。

Low级别

源码审计

  • 没有对user、password进行过滤，导致有注入风险
  • 登录失败没有其他措施，可高速暴力破解
  • 可无限次数暴力猜测密码

解答

解法一：暴力破解

  • 第一步抓包
  • 将包复制到intrude模块
  • payloads 导入字典
  • 开始暴力破解

最终:得到密码为 password，手动验证ok

Medium级别

源码审计

  • 对user、password进行过滤，一定程度上解决注入的风险(mysqli_real_escape_string有一定绕过风险)
  • 登录失败有 sleep(2) 措施，算不上防止暴力破解的方法，只不过每一次失败请求都时间长一点而已
  • 和low等级一样可无限次数暴力猜测密码

High级别

源码审计

  • 使用了stripslashes（去除字符串中的反斜线字符,如果有两个连续的反斜线,则只去掉一个），进一步防御sql注入问题
  • 增加了token的验证，需要效验 user_token 以及 session_token，这两个token是通过 index.php 得到的
  • 失败时随机 sleep 时间，对防暴力破解实质也没什么大用

思路如下:

  • 每次先请求拿到 并解析拿到token
  • 下次请求带着 token， 别且这次请求返回的token作为下次请求的token
  • 绕过token检测机制，实现暴力破解
  • 请求可以走代理，方便查看请求信息

Impossible级别

欣赏源码

  • 设置最大失败次数

  • 设置失败次数过多导致账号锁定后限制请求频率
